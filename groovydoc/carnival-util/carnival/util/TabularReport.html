            • @groovy.transform.Synchronized java.lang.Object start()

              Start must be called before dataAddAttributes() is called. dataAddAttributes() will link data only to identifiers already present in the underlying generic data table. This seems like a half-baked idea, but I think adding a start() method helped by splitting the phases of interraction with TabularReport objects. Before start() the data are built out vertically, with new identifiers being added. After start(), the data can be built out horizontally via dataAddAttributes().
